# 回归检查清单（macOS）

最后更新：2026-04-07

## 一、启动与窗口行为

1. 执行 `npm run tauri dev`，应用可正常启动。
2. 点击窗口关闭按钮，窗口应隐藏而非退出进程。
3. 点击菜单栏托盘图标，窗口可在显示/隐藏间切换。
4. 托盘菜单“显示/隐藏窗口”可正常切换窗口。
5. 托盘菜单“退出”可完全退出应用。

## 二、应用与快捷键 CRUD

1. 新建应用（名称必填），成功后列表即时更新。
2. 编辑应用名称和说明，保存后刷新仍保留。
3. 删除应用时，其快捷键应被级联删除。
4. 新建快捷键（名称、组合键必填）成功。
5. 编辑快捷键组合和备注成功。
6. 删除快捷键成功并即时刷新列表。

## 三、快捷键录入

1. 在“监听输入”模式，按 `Command+Shift+K`，组合键正确显示。
2. 在“下拉选择”模式，选修饰键和主键后组合键正确生成。
3. 仅按修饰键不会保存为合法组合键。
4. 相同应用下重复组合键应被阻止并提示错误。

## 四、全局快捷键设置

1. 设置页能读取当前全局唤起快捷键。
2. 修改后保存成功，顶部提示更新。
3. 新快捷键在应用非激活状态下仍能唤起/收起窗口。
4. 退出并重启应用后，快捷键配置保持不变。
5. 设置一个冲突快捷键时，保存应失败并给出错误提示。

## 五、搜索与筛选

1. 应用搜索框可按名称/说明过滤列表。
2. 快捷键搜索框可按名称/组合键/备注过滤列表。
3. 无匹配时显示“没有匹配到...”提示。

## 六、构建与测试

1. `npm run verify` 全流程通过。
2. 产物存在：
- `src-tauri/target/debug/bundle/macos/Shortcut Key Map.app`

## 七、已知限制

1. 默认打包目标为 `app`，未默认启用 `dmg`。
2. 若需要 `dmg`，需在发布环境单独启用并处理 `hdiutil` 兼容问题。
