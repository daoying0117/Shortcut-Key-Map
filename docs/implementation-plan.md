# 快捷键记忆工具实施计划（Tauri 2 + Vue 3 + SQLite）

## 里程碑 0：基础环境与工程结构
- 目标：建立可运行的 Tauri 2 + Vue 3 + npm 项目骨架，替换当前的纯 Rust hello world。
- 交付物：
- `package.json`、`vite.config.*`、Vue 3 前端目录。
- `src-tauri/`（Tauri 2 Rust 后端）。
- 能运行的 `npm run tauri dev`。
- 验收标准：
- 本地执行 `npm install` 无报错。
- 本地执行 `npm run tauri dev` 能打开窗口并显示 Vue 页面。
- 风险与控制：
- 网络或镜像导致 npm 包下载失败；记录失败日志并切换镜像后重试。

## 里程碑 1：数据模型与 SQLite 持久化
- 目标：完成数据库表设计与基础 CRUD 命令。
- 交付物：
- SQLite 数据库初始化逻辑（首次启动自动建表）。
- 表结构：`apps`、`shortcuts`（含唯一约束和外键）。
- Tauri command：新增/编辑/删除/查询软件与快捷键。
- 验收标准：
- 通过命令可创建软件与快捷键并持久化到本地 sqlite 文件。
- 重启应用后数据仍可读取。
- 错误路径（重复快捷键、空名称）有明确错误信息。
- 风险与控制：
- 模型变更导致迁移复杂；从 v1 起就引入 schema version 和迁移入口。

## 里程碑 2：前端信息架构与 CRUD 页面
- 目标：完成核心业务界面，支持快捷键记录管理。
- 交付物：
- 页面布局：左侧软件列表，右侧快捷键列表/编辑区。
- 功能：软件与快捷键的增删改查、基础筛选与排序。
- 前后端调用封装（TypeScript API 层）。
- 验收标准：
- 从 UI 可完整执行一条“新增软件 -> 新增快捷键 -> 编辑 -> 删除”流程。
- 所有操作能即时刷新 UI 且与数据库数据一致。
- 风险与控制：
- 状态管理复杂度上升；使用明确的 store 分层（view state / server state）。

## 里程碑 3：快捷键录入能力（下拉 + 键盘监听）
- 目标：实现两种录入方式，统一输出规范化按键组合。
- 交付物：
- 下拉方式：修饰键（Cmd/Ctrl/Alt/Shift）+ 主键选择。
- 监听方式：聚焦输入框后捕获组合键（含 mac 键位映射）。
- 规范化算法：组合键排序与去重（如 `Cmd+Shift+K`）。
- 验收标准：
- 同一组合键不同输入顺序得到相同存储结果。
- 能正确识别 mac 常见组合键并展示友好标签。
- 禁止存储非法输入（仅修饰键、空组合、重复键）。
- 风险与控制：
- 键盘事件跨平台差异；先以 mac 为主实现并保留平台分支。

## 里程碑 4：macOS 导航栏常驻与全局快捷键切换
- 目标：应用可常驻上方菜单栏，并通过全局快捷键展开/收起主窗口。
- 交付物：
- 系统托盘（菜单栏图标）与菜单项（显示/隐藏/退出）。
- 全局快捷键注册（默认示例：`Cmd+Shift+K`，可后续配置化）。
- 窗口显示状态切换逻辑（show/hide/focus）。
- 验收标准：
- 关闭窗口后应用仍驻留在菜单栏。
- 快捷键在应用未激活时仍可唤起/隐藏窗口。
- 菜单栏项与快捷键行为一致，不出现窗口状态错乱。
- 风险与控制：
- 全局快捷键冲突；启动时检测并给出可读提示。

## 里程碑 5：设置、稳定性与错误处理
- 目标：补齐可用性细节，提升稳定性。
- 交付物：
- 设置页（默认快捷键、开机启动预留、主题预留）。
- 统一错误提示与日志（前端 toast + Rust 侧 error enum）。
- 基础防呆（重复提交、并发写入保护）。
- 验收标准：
- 常见错误都有可解释提示，不出现静默失败。
- 高频操作（快速新增/删除）不会导致 UI 与 DB 不一致。
- 风险与控制：
- 异步竞态；关键写操作串行化并增加操作后重拉取校验。

## 里程碑 6：测试、打包与发布准备
- 目标：形成可交付的 macOS 安装包与回归用例。
- 交付物：
- 测试清单：手工回归脚本 + 关键单元测试（按键规范化、DB CRUD）。
- `npm run tauri build` 产物（`.app` 或 `.dmg`）。
- 发布说明（已知限制、默认快捷键、数据存储路径）。
- 验收标准：
- 构建成功，安装后可运行核心流程。
- 回归清单全通过后再进入发布。
- 风险与控制：
- 签名/公证配置耗时；先完成开发态可运行版本，再处理正式分发链路。

## 执行顺序与门禁
- 按里程碑 0 -> 6 串行推进。
- 每个里程碑结束后必须先通过“验收标准”，再进入下一阶段。
- 任一阶段失败，不继续叠加功能，先修复并复验。
